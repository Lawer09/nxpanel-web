# v1.4.5

## 优化功能

- 新增用户管理中的 `IP 白名单` 与 `白名单策略` 维护入口：支持白名单 IP 列表查询、批量录入与删除，以及按国家 / 项目代号 / 包名配置自动白名单规则（`src/pages/user-manage/index.tsx`, `src/pages/user-manage/components/AllowedIpModal.tsx`, `src/pages/user-manage/components/IpAllowlistRuleModal.tsx`, `src/pages/user-manage/components/IpAllowlistRuleFormModal.tsx`, `src/services/user/api.ts`, `docs/api/user_api.md`）
- 优化用户管理中的封禁 IP 弹窗：新增“批量封禁 IP”入口，支持粘贴多个 IP 并按空白字符或逗号分隔，同时可设置类型、封禁原因和是否同步封禁命中用户（`src/pages/user-manage/components/BlockedIpModal.tsx`, `src/services/user/api.ts`, `docs/api/user_api.md`）
- 优化用户管理筛选栏：新增注册国家和注册 IP 的可输入下拉筛选，并将筛选值同步展示在当前结果摘要中（`src/pages/user-manage/index.tsx`, `docs/issue/user-manage.md`）
- 优化用户管理中的封禁 IP 列表：新增 `type` 展示列和“更新类型”操作，接入 `/v3/user/blockedIp/updateType` 接口，并将类型更新收口为 `dangerous`、`normal` 下拉选择（`src/pages/user-manage/components/BlockedIpModal.tsx`, `src/services/user/api.ts`, `docs/api/user_api.md`）
- 项目日报与项目小时汇总的项目代号筛选改为进页一次加载项目列表，输入时仅做前端本地过滤，不再触发 `/projects?keyword=...` 远程搜索并刷新选项（`src/pages/report/project/index.tsx`, `src/pages/report/project-hourly/index.tsx`）
- 优化项目日报、项目小时汇总与项目趋势 Dashboard 的项目代号下拉数据源：筛选候选统一切换为 `/v3/projects/project-codes`，不再复用项目列表接口拉取整页项目数据（`src/services/project/api.ts`, `src/pages/report/project/index.tsx`, `src/pages/report/project-hourly/index.tsx`, `src/pages/report/project-trend/index.tsx`, `docs/api/project_api.md`）
- 优化标准日期快捷筛选项：在通用日期预设中新增“上一周”“上一月”，供项目日报、项目小时汇总、项目趋势 Dashboard 等复用标准预设的页面统一使用（`src/components/report/reportDatePreset.ts`）
- 优化项目日报与项目小时汇总筛选能力：新增 `departments` 部门筛选，沿用投放状态同级的前端筛选模式，候选值复用项目部门列表接口并支持手动输入（`src/pages/report/project/index.tsx`, `src/pages/report/project-hourly/index.tsx`, `src/services/report/typings.d.ts`）
- 优化项目日报与项目小时汇总的总支出列展示：在总支出数值下补充“投放支出 / 流量支出”占比条，悬浮后可查看两部分成本占比（`src/pages/report/project/reportShared.tsx`）
- 优化项目管理粘贴导入：新增对表格剪贴板 TSV 文本的识别，支持制表符分列和单元格内换行续接，避免从表格复制整行项目数据时被误判为“未识别到可导入记录”（`src/pages/project-table/pasteImport.ts`, `docs/issue/project_table.md`）
- 优化项目管理导入弹窗：CSV 字段映射支持同一表头重复选择并展示已选次数，粘贴导入支持多个空格分隔字段，且缺少项目名称时默认回填项目代号、缺少负责人或包名降级为警告后继续导入（`src/pages/project-table/components/ProjectBatchImportModal.tsx`, `src/pages/project-table/pasteImport.ts`, `docs/issue/project_table.md`）
- 优化项目趋势 Dashboard 默认时间与触发方式：移除跳转兜底回退逻辑，日模式默认最近 7 天、小时模式默认今天，且仅 `日 / 小时` 粒度切换会立即刷新，其它筛选继续通过“查询”按钮生效（`src/pages/report/project-trend/index.tsx`, `src/pages/report/project-trend/utils.ts`, `docs/components/project_trend_dashboard.md`）
- 优化项目趋势 Dashboard 小时模式筛选交互：小时粒度下将日期控件收口为带小时的时间范围选择器，支持直接选择跨天小时区间，如“昨日 08:00 到今日 06:00”（`src/pages/report/project-trend/index.tsx`, `src/pages/report/project-trend/components/TrendDashboardHeader.tsx`, `src/pages/report/project-trend/utils.ts`）
- 优化项目趋势 Dashboard 查询触发方式：页面内部拆分 `draftQuery / appliedQuery`，筛选修改后不再自动刷新，点击“查询”后才发起请求并同步 URL（`src/pages/report/project-trend/index.tsx`, `src/pages/report/project-trend/components/TrendDashboardHeader.tsx`）
- 优化项目趋势 Dashboard 小时模式图表展示：切换到小时粒度后隐藏“广告收益对比趋势”，避免小时口径下重复展示不适合的对比图（`src/pages/report/project-trend/index.tsx`）
- 优化项目日报与项目小时汇总的广告收入列展示：若接口返回 `topRevenueCountries`，单元格中补充展示最高收益国家，悬浮后可查看完整国家收益与占比列表（`src/pages/report/project/reportShared.tsx`, `docs/api/project_report_query_api.md`, `docs/api/project_hourly_report_query_api.md`）
- 优化项目日报与项目小时汇总的限流提示：若接口返回 `recentHourlyAdMatchRates`，在 `限流` Tag 悬浮层中补充最近 12 小时广告匹配率迷你图，并绘制 `70%` 参考线辅助判断（`src/pages/report/project/reportShared.tsx`, `docs/api/project_report_query_api.md`, `docs/api/project_hourly_report_query_api.md`）
- 优化项目日报与项目小时汇总的广告收入国家分布展示：移除 `TOP` 文字和金额展示，改为仅按占比渲染国家迷你条图，悬浮后查看完整国家占比列表（`src/pages/report/project/reportShared.tsx`）
- 优化项目日报与项目小时汇总的广告收入国家分布图例：将前三国家合并为单条分段占比条，`US` 固定蓝色、其他 Top 国家使用非蓝色，剩余非 Top3 占比统一为灰色，并移除单元格内国家文案（`src/pages/report/project/reportShared.tsx`）
- 优化项目趋势 Dashboard 文档可维护性：重写趋势页组件说明与问题记录，补充小时模式和手动查询的约束（`docs/components/project_trend_dashboard.md`, `docs/issue/project_trend_dashboard.md`）
- 优化 Firebase 节点状态列表交互：移除单独暴露的 `诊断状态 / 样本范围` 筛选，列表只保留快捷问题视图与成功率、样本、P95、错误等核心排查字段，减少无意义口径噪音（`src/pages/firebase-analytics/NodeStatus.tsx`）
- 优化 Firebase 节点状态详情分析能力：详情页新增统一筛选栏，并补充国家成功率分布、连接/探测成功率趋势、连接错误占比和探测错误占比图，支持从节点状态列表携带参数下钻分析（`src/pages/firebase-analytics/NodeStatusDetail.tsx`）
- 优化 Firebase 通用筛选栏交互：App、平台、版本、国家、网络、ISP、ASN、节点国家/区域、协议等下拉项改为“可选现有值，也可直接输入”，避免筛选值受限于接口返回候选项（`src/components/FirebaseAnalytics/FilterBar.tsx`）
- 优化 Firebase 通用筛选栏触发方式：筛选项与快捷时间按钮改为先编辑草稿，点击“查询”后再统一生效，避免选择过程中自动请求（`src/components/FirebaseAnalytics/FilterBar.tsx`）

## Bug 修复

- 修复报表日期快捷筛选“今日 / 昨日 / 近一周”等相对日期在页面长时间停留后仍使用首次加载日期的问题：`RangePicker.presets` 改为读取时动态重算，避免 7 月 8 日点击“今日”仍落到 7 月 6 日（`src/components/report/reportDatePreset.ts`）
- 修复封禁 IP 列表中 `operator_user` 文案误显示为“管理员”的问题：关联用户区第二行统一更正为“操作人”（`src/pages/user-manage/components/BlockedIpModal.tsx`, `docs/issue/user-manage.md`）
- 修复用户管理单条“解封”仍误调用封禁接口的问题：用户手动解封改为走 `/v3/user/update`，将 `banned` 更新为 `false`，并明确删除封禁 IP 记录不会自动解除用户封禁（`src/pages/user-manage/index.tsx`, `docs/api/user_api.md`, `docs/issue/user-manage.md`）
- 修复项目日报与项目小时汇总在刷新报表后又额外触发 `/projects` 初始化请求的问题：项目代号选项改为页面生命周期内仅加载一次，避免刷新报表时重复拉取项目列表（`src/pages/report/project/index.tsx`, `src/pages/report/project-hourly/index.tsx`）
- 修复 Firebase 节点状态详情进入后无法继续沿用时间、App、平台、国家等筛选上下文的问题，详情页现已复用 Firebase 通用筛选参数工具并保持返回列表链路不丢失（`src/pages/firebase-analytics/NodeStatusDetail.tsx`, `src/utils/firebase-analytics-filters.ts, docs/issue/firebase_analytics.md`）
