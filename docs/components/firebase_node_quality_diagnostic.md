# Firebase 节点质量诊断面板

> 说明：`NodeQualityTable` 为旧版 Dashboard 深度诊断组件。当前 Firebase 节点排查主入口已迁移到 `节点状态` 菜单，Dashboard 只保留摘要入口。

## 组件名称

`NodeQualityTable`

## 组件用途

在旧版 Firebase Dashboard 中对照节点探测样本与真实连接样本，优先识别以下几类节点问题：

- 仅探测未连接
- 探测正常但连接偏差
- 探测与连接双侧高风险
- 连接有样本但缺少探测覆盖

该组件不只是展示排行，而是服务于节点质量排查。

## 使用场景

- 维护旧版 Dashboard 节点诊断实现时参考
- 需要补充“节点仅被测速、没有真实连接样本”的观测口径
- 需要在一个面板内同时查看连接排行、探测统计和诊断结论

## Props 说明

| Props | 类型 | 说明 |
|---|---|---|
| `filters` | `FirebaseAnalyticsFilter` | Dashboard 当前筛选条件，会同时透传给连接排行接口和探测统计接口 |

## 数据来源

- `/v3/firebase-analytics/nodes/quality-rank`
  - 读取 `source=session` 的真实连接聚合数据
- `/v3/firebase-analytics/vpn-probe/node-stats`
  - 读取节点探测统计数据

## 诊断口径

- `仅探测未连接`
  - 有探测样本，但没有真实连接样本
- `探测正常但连接偏差`
  - 探测成功率较高，且明显高于真实连接成功率
- `探测与连接双侧风险`
  - 探测和连接任一侧都已进入风险区间
- `连接侧风险`
  - 真实连接成功率或连接 P95 耗时异常
- `探测侧风险`
  - 探测成功率或探测 P95 延迟异常
- `连接有样本但缺少探测`
  - 有真实连接数据，但没有探测覆盖

## 示例代码

```tsx
<NodeQualityTable filters={apiParams} />
```

## 注意事项

- 节点对齐使用 `node_id + protocol + node_country` 组合键，避免仅用名称匹配导致混淆。
- 诊断结果依赖“探测口径”和“真实连接口径”的并排比较，不应退化为只展示任一侧排行。
- 当后端后续补充更稳定的节点唯一键或 mixed 诊断接口时，应优先复用后端口径，而不是继续在前端堆规则。

## 已知限制

- 当前“探测正常但连接偏差”仍基于前端阈值规则判断，不是后端专门诊断接口。
- 如果某些节点在探测与连接接口中的标识方式不完全一致，仍可能出现样本无法合并的问题，需要后端进一步统一节点维度。
- 新功能优先接入 `节点状态` 页面，不建议继续扩展 `NodeQualityTable` 承担列表筛查和详情排查职责。
