请设计并实现一个“自动化控制台弹窗”，只关注主内容区，不需要顶部导航和左侧系统 Sidebar。

该弹窗不是单纯的规则配置表单，而是一个小型控制台，用于管理某个模块下的自动化规则、运行状态、执行记录、异常记录、手动执行和规则编辑。

整体要求：
1. 使用超大弹窗或全屏弹窗，宽度约 1180px-1360px，高度约 86vh-92vh。
2. 弹窗标题为“自动化控制台”。
3. 标题下方展示当前 module，例如 traffic_platform，以及默认 targetType，例如 traffic_platform_account。
4. Header 右侧提供操作按钮：刷新、Dry Run 全部、新建规则。
5. 弹窗内部顶部展示状态概览卡片：
   - 规则总数
   - 已启用规则
   - 最近 24 小时触发次数
   - 失败次数
   - 最近执行时间
6. 主体使用左右分栏布局：
   - 左侧宽度约 360px-420px
   - 右侧自适应占满剩余空间
7. 左侧是控制台导航区，包含 Tabs：
   - 规则
   - 执行记录
   - 异常
8. 左侧“规则”Tab 中使用卡片列表展示规则，不要使用传统表格。
9. 每个规则卡片展示：
   - 规则名称
   - enabled 状态 Tag
   - targetType
   - 条件摘要，例如 balance_mb <= 1024
   - 动作摘要，例如 Telegram 管理员通知
   - 冷却时间
   - 最近执行时间
   - 快捷操作：运行、Dry Run、编辑
10. 左侧“执行记录”Tab 展示最近执行记录卡片，包含：
   - status
   - ruleName
   - targetName
   - executedAt
   - matchedConditions 摘要
11. 左侧“异常”Tab 只展示 status=failed 的执行记录。
12. 右侧是选中规则的详情工作区。
13. 右侧顶部展示：
   - 规则名称
   - 描述
   - 启用 Switch
   - 状态标签，例如 最近触发、最近失败、冷却中
14. 右侧内部使用 Tabs：
   - 概览
   - 执行记录
   - 配置
   - 测试
15. “概览”Tab 展示：
   - 规则摘要
   - 目标范围
   - 触发条件
   - 执行动作
   - 冷却与恢复
   - 最近执行记录
16. “执行记录”Tab 展示当前规则的执行记录，支持按状态过滤。
17. “配置”Tab 是规则编辑表单，但不要使用多步骤向导，而是使用分组 Card：
   - 基础信息
   - 目标范围
   - 触发条件
   - 执行动作
   - 冷却与恢复
18. “测试”Tab 提供手动执行控制台：
   - Dry Run / 真实执行切换
   - 目标对象选择
   - 运行按钮
   - 执行结果展示
19. 当动作包含 disable_account，真实执行前必须展示高风险提示和二次确认。
20. Footer 固定在底部，提供：
   - 关闭
   - 保存配置
   - 保存并启用
   - 运行 Dry Run

接口依据：
- 查询模型：GET /api/v3/admin/{securePath}/automation-rules/models?module=traffic_platform
- 规则列表：GET /api/v3/admin/{securePath}/automation-rules?module=traffic_platform
- 规则详情：GET /api/v3/admin/{securePath}/automation-rules/detail?id=1&module=traffic_platform
- 创建规则：POST /api/v3/admin/{securePath}/automation-rules/create
- 更新规则：POST /api/v3/admin/{securePath}/automation-rules/update
- 更新状态：POST /api/v3/admin/{securePath}/automation-rules/update-status
- 手动执行：POST /api/v3/admin/{securePath}/automation-rules/run
- 执行记录：GET /api/v3/admin/{securePath}/automation-rules/executions?module=traffic_platform

traffic_platform 模块字段：
- targetType: traffic_platform_account
- targetScope:
  - platformCodes: string[]
  - accountIds: int[]
  - includeDisabled: 0|1
- conditions metric:
  - balance_mb
  - usage_1h_mb
  - usage_6h_mb
  - avg_hourly_usage_mb
  - eta_hours
  - last_sync_minutes
  - enabled
  - account_id
  - account_name
  - platform_code
- actions type:
  - telegram_admin
  - email
  - disable_account

视觉风格：
- 后台控制台风格
- 深浅层次清晰
- 卡片化布局
- 左侧列表密度适中
- 右侧详情信息完整
- 状态颜色明确：
  - 启用：绿色
  - 停用：灰色
  - 触发：橙色
  - 恢复：绿色
  - 失败：红色
  - Dry Run：蓝色
  - 高风险动作：红色
- 弹窗内部允许滚动，但 Header、顶部概览和 Footer 尽量固定。
- 不要设计成普通 CRUD 表格。
- 不要把所有内容堆进一个表单。
- 要让用户打开弹窗后立刻知道：
  1. 当前有多少规则
  2. 哪些规则正在运行
  3. 最近有没有触发
  4. 有没有失败
  5. 可以快速执行、测试、编辑、停用规则
